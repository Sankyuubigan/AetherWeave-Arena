# [Название Игры] - Project Overview

## 1. Философия и цели проекта
1. Выразить творчество и поделиться уникальными игровыми идеями.
2. Создать платформу для других людей, чтобы они выражали свои идеи.
3. Исправить недостатки современных игр: убрать плоские механики, Pay-to-Win и отсутствие баланса.

## 2. Глобальное видение (Vision)
* **Жанр:** 3D Action от третьего лица (PvP + PvE).
* **Боевка:** Non-target, магия, динамика.
* **Язык разработки:** Строго **GDScript** (Godot 4).
* **Сетевая модель (Этап 1 - P2P):** Режим "Игрок-Хост" (Listen Server). Игроки могут сами создавать лобби бесплатно.
* **Сетевая модель (Этап 2 - Dedicated):** Когда игра масштабируется (например, клановые бои на 32 человека), проект компилируется в режиме "Без визуала" (Headless) и выкладывается в открытый доступ, чтобы комьюнити могло запускать выделенные сервера на мощных машинах.

## 3. Сетевые парадигмы (Строгие правила)
Игра строится на трех китах киберспортивного мультиплеера:
1. **Authoritative Server (Авторитетный сервер):** Сервер никому не верит. Клиент только отправляет инпуты (нажатия клавиш). Сервер решает, что произошло, и рассылает результат всем.
2. **Client-Side Prediction (Предсказание на клиенте):** Чтобы игра не казалась "ватной", клиент локально предсказывает свои движения и действия ДО ответа сервера. Если сервер присылает другой результат, клиент жестко корректируется (Rubberbanding).
3. **Lag Compensation (Компенсация пинга):** Сервер хранит историю позиций хитбоксов за последние 200-250 мс. При получении выстрела от клиента, сервер "отматывает" время назад для проверки попадания. Максимальная компенсация — 200 мс (Window of Acceptability). Опоздавшие пакеты удаляются.
